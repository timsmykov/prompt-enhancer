## Documentation Standards

This project follows these documentation standards:

1. **Clear Structure** - Consistent formatting with headers, lists, and code blocks
2. **Up-to-Date** - Current status and changelog reflect reality
3. **Actionable** - Next steps and priorities clearly defined
4. **Historical Context** - Archive preserves decision-making history
5. **Quick References** - Dashboards and summaries for fast access

## Contributing to Documentation

When updating documentation:

1. **Keep it Current** - Update current_status.md when making changes
2. **Document Changes** - Add entries to changelog.md
3. **Archive Old Docs** - Move dated reports to archive/ folder
4. **Use Clear Language** - Avoid jargon, explain technical terms
5. **Include Examples** - Code snippets and commands where helpful
